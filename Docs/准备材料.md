我来帮您整理从税务系统数据生成T3凭证模板所需的准备工作。

## 一、税务系统数据准备清单

### 1. 基础凭证信息（必需）
| 字段 | 税务系统来源 | 说明 |
|------|-------------|------|
| **制单日期** | 税款所属期/申报日期 | 凭证日期 |
| **会计期间** | 税款所属期月份 | 如2026-03 |
| **凭证类别字** | 固定为"记"或"付" | 税务相关凭证通常用"记" |
| **凭证编号** | 系统自动生成或手动编排 | 确保不重复 |
| **附单据数** | 申报表数量 | 如1张增值税申报表 |
| **摘要** | 税种+所属期 | 如"3月增值税申报" |

### 2. 科目与金额信息（核心）
| 字段 | 税务系统来源 | 示例 |
|------|-------------|------|
| **科目编码** | 需对照目标账套科目表 | 如22210101（应交增值税-销项税额） |
| **借方金额** | 应纳税额/已缴税额 | 根据业务判断借贷方向 |
| **贷方金额** | 应纳税额/已缴税额 | 同上 |
| **币种** | 固定为"人民币"或空 | 默认本位币 |

### 3. 辅助核算信息（如有）
| 字段 | 税务系统来源 | 说明 |
|------|-------------|------|
| **部门编码** | 税务申报所属部门 | 需匹配目标账套部门档案 |
| **项目编码** | 特定项目税款 | 如工程项目相关税费 |
| **客户编码/供应商编码** | 发票对方单位 | 需提前在T3建立档案 |

## 二、目标账套准备工作

### 1. 科目表对照（关键！）
```
税务系统税种 → T3科目编码对照示例：
- 增值税销项税额 → 22210101
- 增值税进项税额 → 22210102  
- 企业所得税 → 222106
- 城建税 → 222108
- 教育费附加 → 222109
- 地方教育附加 → 222110
- 印花税 → 222111
```

**建议**：先打印目标账套的科目表，建立税务系统税种与T3科目编码的**对照表**。

### 2. 辅助核算档案检查
- **客户档案**：税务系统中的购方/销方单位名称需在T3客户档案中存在，且编码一致
- **供应商档案**：同理
- **部门档案**：如有按部门核算税费，需确保部门编码一致
- **项目档案**：如有按项目核算，需提前建立项目目录

### 3. 凭证类别设置确认
- 登录T3 → 基础设置 → 财务 → 凭证类别
- 确认有"记账凭证"（类别字：记），这是税务凭证最常用的类型

## 三、数据转换工具准备

### 方案A：Excel公式处理（适合数据量小）
1. 税务系统导出Excel
2. 新建对照表页签（税种↔科目编码）
3. 用VLOOKUP匹配科目编码
4. 按凭证模板格式整理列顺序

### 方案B：Python自动化（适合数据量大/定期处理）
需要准备的映射文件：
- `科目对照表.json`：税种名称→科目编码
- `客户编码对照表.json`：税务系统单位名称→T3客户编码
- `部门编码对照表.json`：如有

## 四、数据校验清单（导入前必做）

| 检查项 | 检查内容 |
|--------|---------|
| 科目编码有效性 | 在目标账套中确实存在 |
| 借贷平衡 | 每张凭证借方合计=贷方合计 |
| 辅助核算完整性 | 科目带辅助核算时，对应字段不能为空 |
| 日期合法性 | 制单日期在会计期间内，且未结账 |
| 编码一致性 | 客户/供应商/部门编码与目标账套完全匹配 |

---

## 五、建议操作流程

```
1. 税务系统导出原始数据
2. 准备科目对照表（税务税种↔T3科目编码）
3. 准备客户/供应商编码对照表（如有）
4. 按凭证模板格式整理数据（可用Excel或Python）
5. 数据校验（重点检查科目编码和借贷平衡）
6. 用T3系统工具导入
7. 导入后在T3中查询凭证并审核记账
```
