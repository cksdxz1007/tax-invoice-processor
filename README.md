# 税务应收应付数据处理系统

自动化处理发票数据，生成应收/应付数据和凭证分录。

## 功能特性

### 发票处理
- 自动拆分发票数据（专票、普票、进项）
- 按单位名称汇总生成应收/应付数据
- 生成符合模板的凭证分录（73字段）
- 自动匹配客户/供应商编码

### 档案管理
- 客户档案维护（增删改查、导入导出）
- 供应商档案维护（增删改查、导入导出）
- 批量导入支持 Excel/CSV

## 项目结构

```
├── app.py                    # Web服务入口

├── data.db                   # SQLite数据库
├── templates/
│   ├── index.html            # 首页
│   └── manage.html           # 档案管理页面

```

```bash
uv pip install flask pandas openpyxl
```

## 使用方式

### Web服务

```bash
python app.py
```

访问 http://127.0.0.1:5000

- 首页：上传发票文件，选择月份，处理后下载结果
- 档案管理：维护客户/供应商档案

### 命令行

```bash
# 处理指定月份
python run_full_pipeline.py 3

# 指定发票文件和输出目录
python run_full_pipeline.py 3 --invoice-file data.xlsx --output-dir output

# 仅生成应收/应付数据
python generate_receivable.py Docs/3月发票明细.xlsx
```

## 数据处理流程

```
原始发票文件 (2026年发票统计.xlsx)
    │
    ▼
┌─────────────────────────────────────────┐
│  1. 拆分发票数据                          │
│     - 专票 → 专票sheet                   │
│     - 普票 → 普票sheet                   │
│     - 进项 → 进项sheet                   │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  2. 生成应收/应付数据                      │
│     - 专票+普票 → 应收数据sheet           │
│     - 进项 → 应付数据sheet                │
└─────────────────────────────────────────┘
    │
    ▼
┌─────────────────────────────────────────┐
│  3. 生成凭证分录                         │
│     每行应收数据拆分为3条分录:            │
│     - 借: 应收账款 (单票合计)             │
│     - 贷: 商品销售收入 (金额)             │
│     - 贷: 销项税 (税额)                  │
└─────────────────────────────────────────┘
```

## 输出文件

| 文件 | 说明 |
|------|------|
| `{月份}月发票明细.xlsx` | 发票明细 + 应收数据 + 应付数据 |
| `{月份}月应收凭证分录.xlsx` | 凭证分录（73字段） |
| `{月份}月未匹配客户.xlsx` | 未匹配客户列表 |

## 数据库

SQLite数据库 `data.db`，包含以下表：

- `customers` - 客户档案
- `suppliers` - 供应商档案
- `accounts` - 会计科目
- `data_dictionary` - 数据字典

## 凭证分录模板

符合标准凭证模板，共73个字段：
- 凭证基本信息（会计期间、凭证编号、日期等）
- 分录信息（摘要、科目、金额）
- 辅助核算（客户、供应商、部门、项目等）
- 控制字段（可修改标志、核销标志等）
